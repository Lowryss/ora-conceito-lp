<task type="auto">
  <name>Task 1: Substituir background do hero e recalibrar overlay</name>
  <files>index.html</files>
  <action>
Localizar o bloco CSS `.hero-bg` (em torno da linha 62–67 no arquivo atual) e substituir as duas propriedades:

ANTES:
```css
.hero-bg {
  position: absolute; inset: 0;
  background:
    linear-gradient(to bottom, rgba(14,14,14,.4) 0%, rgba(14,14,14,.72) 70%, rgba(14,14,14,1) 100%),
    url('assets/logo-bg-preta.jpg') center/cover no-repeat;
}
```

DEPOIS:
```css
.hero-bg {
  position: absolute;
  top: -20%; bottom: -20%; left: 0; right: 0;
  background:
    linear-gradient(to bottom,
      rgba(14,14,14,.55) 0%,
      rgba(14,14,14,.65) 60%,
      rgba(14,14,14,1)  100%
    ),
    url('assets/foto-21.png') center/cover no-repeat;
  will-change: transform;
}
```

Mudanças:
- `inset: 0` → `top: -20%; bottom: -20%; left: 0; right: 0` — margem extra para parallax (com translateY o bg não descobre bordas)
- `logo-bg-preta.jpg` → `foto-21.png` — foto de ambiente real
- Overlay no topo: `.4` → `.55` — cobertura mais densa para foto com conteúdo claro/colorido
- Overlay no meio: `.72` → `.65` — levemente menos denso no meio (gradiente mais natural)
- Adicionar `will-change: transform` — hinting de performance para o compositor do browser

NÃO remover `overflow: hidden` do `#hero` — o parallax depende disso para não vazar visualmente.
