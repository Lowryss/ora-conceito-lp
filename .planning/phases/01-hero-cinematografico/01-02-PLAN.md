  <action>
Localizar o bloco `<script>` ao final do arquivo (em torno da linha 907). O script atual termina com o IntersectionObserver (`revealEls.forEach(el => observer.observe(el));`).

Adicionar o bloco de parallax APÓS o último statement do script existente, antes do `</script>`:

```javascript
  // Hero parallax
  const heroBg = document.querySelector('.hero-bg');
  const noMotion = window.matchMedia('(prefers-reduced-motion: reduce)');
  const isTouch  = window.matchMedia('(hover: none)');

  let ticking = false;
  window.addEventListener('scroll', () => {
    if (!ticking) {
      requestAnimationFrame(() => {
        if (!noMotion.matches && !isTouch.matches) {
          heroBg.style.transform = 'translateY(' + (window.scrollY * 0.4) + 'px)';
        }
        ticking = false;
      });
      ticking = true;
    }
  }, { passive: true });
```

Detalhes críticos:
- Fator `0.4` — 40% da velocidade do scroll. Sutil mas perceptível.
- Padrão `ticking` — evita empilhar múltiplos rAF por evento de scroll (throttle nativo 60fps)
- `passive: true` — obrigatório para não bloquear o thread principal (já usado no nav scroll existente)
- `prefers-reduced-motion` — respeitar preferência de acessibilidade
- `hover: none` — desativa parallax em touch (cobre iOS Safari onde background-attachment:fixed não funciona)
- NÃO usar `background-attachment: fixed` em lugar algum no .hero-bg — é o anti-pattern documentado na research

O `#hero` pai já tem `overflow: hidden` — não remover isso.
